学习数据挖掘的最佳路径是什么（笔记）

发表于 2020-01-22 分类于 Data Analysis Disqus：
本文字数： 1.3k 阅读时长 ≈ 1 分钟

数据挖掘知识清单

数据挖掘的基本流程

  • 商业理解
    • 从商业的角度理解项目需求
    • 对数据挖掘的目标进行定义
  • 数据理解
    • 收集部分数据
    • 对数据进行探索（包括数据描述、数据质量验证等）
  • 数据准备
    • 开始收集数据
    • 数据清洗、数据集成等
  • 模型建立
    • 选择和应用各种数据挖掘模型
    • 进行优化从而得到更好的结果
  • 模型评估
    • 对模型进行评价
    • 检测构建模型的每个步骤
    • 确认模型是否实现了预定的商业目标
  • 上线发布
    • 将从数据中找到的“知识”转化为用户可以使用的方式
    • 呈现的形式可以是报告或者是可重复的数据挖掘过程
    • 后续的监控和维护对数据挖掘结果的日常运营很重要

数据挖掘的十大算法

  • 分类算法
    • C4.5
      • 决策树算法
      • 创造性地在决策树构造过程中就进行了剪枝
      • 可以处理连续的属性和不完整的数据
    • 朴素贝叶斯（Naive Bayes）
      • 基于概率论的原理
      • 求解在未知物体出现的条件下各个类别出现的概率
      • 哪个最大，就认为这个未知物体属于哪个分类
    • SVM
      • 支持向量机（Support Vector Machine）
      • 在训练中建立超平面
    • KNN
      • K 最近邻算法（K-Nearest Neighbor）
      • K 近邻即每个样本都可以用它最接近的 K 个邻居来代表
    • AdaBoost
      • boost 在英文中代表提升的意思，所以 Adaboost 是个构建分类器的提升算法
      • 在训练中建立一个联合的分类模型
      • 让多个弱的分类器组成一个强的分类器
    • CART
      • 分类和回归树（Classification and Regression Trees）
      • 构建了一棵分类树和一棵回归树
      • 属于决策树学习方法
  • 聚类算法
    • K-Means
      • K 个类别，每个类别设置“中心点”
      • 计算新点与 K 个中心点的距离
      • 距离哪个中心点近就划分为该类
    • EM
      • 最大期望算法，用于求参数的最大似然估计
      • 假设 A 和 B 未知，A 和 B 都可以通过对方的值得到自己的估值
      • 可以给其中一方赋初值，得到另一方的估值，然后反过来求取，持续这个过程直到收敛为止
  • 关联分析
    • Apriori
      • 是一种挖掘关联规则（association rules）的算法
      • 通过挖掘频繁项集（frequent item sets）来揭示物品之间的关联关系
      • 频繁项集指经常出现在一起的物品的集合
      • 关联规则暗示着两种物品之间可能存在很强的关系
      • 被广泛应用到商业挖掘和网络安全等领域中
  • 连接分析
    • PageRank
      • 当一个页面链出的页面越多，说明这个页面的“参考文献”越多
      • 当一个页面被链入的频率越高，说明这个页面被引用的次数越高
      • 基于这个原理可以得到网站的权重划分

数据挖掘的数学原理

  • 概率论与数理统计
    • 条件概率
    • 独立性
    • 随机变量
    • 多维随机变量
  • 线性代数
    • 特征值和特征向量
    • 用特征向量近似代表物体的特征（大数据降维的基本思路）
    • 基于矩阵的运算和理论
      • PCA 方法
      • SVD 方法
      • MF 方法
      • NMF 方法
  • 图论
    • 起源自社交网络的好友关系表达
    • 人与人的关系，可以用图论上的两个节点来进行连接
    • 节点的度可以理解为一个人的朋友数
    • 图论对于网络结构的分析非常有效
    • 图论在关系挖掘和图像分割中有重要的作用
  • 最优化方法
    • 相当于机器学习中自我学习的过程
    • 当机器知道了目标，训练后与结果存在偏差就需要迭代调整，最优化就是这个调整的过程
    • 一般来说，这个学习和迭代的过程是漫长、随机的
    • 最优化方法的提出就是用更短的时间得到收敛，取得更好的效果
